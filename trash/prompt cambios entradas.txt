1. VISIÓN DEL PROYECTO
   ────────────────────────────
   Sistema de venta de entradas y consumo sin fila para eventos.
   No es solo “barra”: es una plataforma escalable para eventos, locales saturados y experiencias tipo McDonald's App / Totem.

Estado actual:

* Funcional
* En producción (Render)
* Con usuarios reales
* Con pagos reales (Mercado Pago y Rapipago / Pagofacil)

Objetivo:
Iterar rápido SIN romper funcionalidades ya validadas.

────────────────────────────
2) STACK TECNOLÓGICO
────────────────────────────

* Backend: FastAPI (monolítico)
* Frontend: Jinja2 + HTML + CSS + JS vanilla
* DB: SQLite (por ahora)
* Hosting: Render
* Pagos: Mercado Pago (Checkout Pro)
* Auth: login propio (cliente / productor / staff)

────────────────────────────

7) OBJETIVO DE ESTE PEDIDO PUNTUAL
────────────────────────────
es un cambio grande el que quiero hacer. en el menú de productor. la lógica de las entradas quiero que sea distinta. no quiero preventas, no quiero consumiciones.
quiero poder cargar como venta cualquier cosa (parking, preventa1, entrada, general, vip, consumición, etc), que el productor pueda determinar q vender, con stock y fecha
por ejemplo : entrada general preventa 1  100 unidades de x fecha a x fecha precio x
entrada vip preventa 1 100 unidades de x fecha a x fecha precio x
también quiero que se puedan cargar vendedores (que compartan su link del evento y esa venta les cuente a ellos, ósea un evento puede tener distintos links de venta de distintos vendedores) llevar un registro de esto también.
que el productor pueda emitir ticket de cortesia. qr que pueda enviar por whatsapp desde el mismo panel.
informes de entradas vendidas, simple pero claro
informe de ventas por vendedor, simple pero claro
todo accesible desde la pc o desde el celular
